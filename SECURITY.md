# Security Policy

## Reporting a Vulnerability

If you discover a security issue in `xlflow`, please email **harumiweb.security@gmail.com** with:

- a description of the issue
- affected files or commands
- steps to reproduce
- any proof of concept, logs, or screenshots

Please do not open a public issue for security-sensitive reports.

## What to Include

The more detail you provide, the faster we can assess and fix the issue. If the report affects a workbook, macro, or command flow, include the exact workbook type, command line, and expected vs. actual behavior.

## Scope

This policy applies to the repository contents, CLI behavior, and any generated artifacts related to `xlflow`.
